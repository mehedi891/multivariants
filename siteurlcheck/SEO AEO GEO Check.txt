Here is a stronger prompt designed for Claude Code to audit the complete website before making any changes:

Act as a senior Technical SEO, AEO, GEO, and AI-discoverability specialist.

Perform a complete end-to-end audit of this website project to identify anything that could prevent the site from:

* Ranking effectively in traditional search engines
* Appearing in featured snippets and answer engines
* Being understood and cited by AI assistants
* Being crawled, indexed, and interpreted correctly
* Establishing clear topical authority, entity relevance, and trust

Use specialized sub-agents where helpful to review the project in parallel, including:

1. Technical SEO and crawlability
2. On-page SEO and metadata
3. Content structure and internal linking
4. Structured data and schema markup
5. AEO and featured-snippet readiness
6. GEO and AI citation readiness
7. Performance, accessibility, and rendering
8. Analytics and search-engine verification setup

## Important instructions

Do not modify any files during this phase.

First inspect the entire project, identify issues, verify each finding against the actual code, and create a prioritized remediation plan for my review.

Do not make assumptions based only on filenames. Trace how pages, layouts, metadata, routes, content, and structured data are actually rendered.

Avoid generic SEO advice. Every finding must be specific to this project and supported by evidence such as:

* File path
* Component or function name
* Route or page
* Relevant code reference
* Rendered output, where it can be verified
* Explanation of the likely SEO or discoverability impact

## Audit scope

### 1. Crawling and indexing

Check:

* robots.txt
* XML sitemap generation and coverage
* Canonical URLs
* Index and noindex directives
* Redirect behavior
* HTTP status handling
* Duplicate and parameterized URLs
* Pagination
* Trailing-slash consistency
* HTTP/HTTPS and www/non-www consistency
* Broken routes and links
* Orphan pages
* JavaScript rendering dependencies
* Whether important content is available in the initial rendered HTML
* Whether search engines can access all important pages and assets
* Whether staging, development, admin, or private routes could be indexed accidentally

### 2. Metadata

Review every indexable page and route for:

* Unique and descriptive meta titles
* Meta description quality
* Title and description length
* Search intent alignment
* Primary keyword placement
* Duplicate or missing metadata
* Metadata inheritance and fallback behavior
* Dynamic metadata generation
* Open Graph metadata
* X/Twitter metadata
* Social preview images
* Canonical metadata
* Robots metadata
* Language and locale declarations
* Alternate-language or hreflang requirements, where applicable

Create a page-level metadata inventory showing the existing title, description, canonical URL, and identified issue.

### 3. Page and heading structure

Check:

* Whether every page has one clear H1
* Logical H1-H6 hierarchy
* Heading duplication
* Headings being used only for styling
* Whether headings clearly communicate the page topic
* Semantic HTML structure
* Landmark elements such as header, nav, main, article, section, aside, and footer
* Descriptive link and button text
* Image alt text
* Anchor text quality
* Content accessibility for crawlers and assistive technologies

### 4. Content quality and search intent

Evaluate whether each major page:

* Clearly explains what the company, product, or service does
* Targets a defined search intent
* Answers the visitor’s main questions
* Includes meaningful benefit-focused content
* Contains sufficient original information
* Avoids thin, repetitive, vague, or keyword-stuffed copy
* Uses consistent terminology
* Demonstrates expertise and trust
* Includes appropriate proof, examples, use cases, FAQs, comparisons, or supporting evidence
* Has a clear relationship with other pages in the site architecture

Identify content gaps and keyword cannibalization risks.

Do not invent search volume or keyword data unless reliable research tools are available.

### 5. Internal linking and information architecture

Review:

* Navigation structure
* Footer links
* Breadcrumbs
* Contextual internal links
* Topic clusters
* Hub and supporting pages
* Anchor-text quality
* Click depth
* Orphaned or weakly connected pages
* Links between product, feature, use-case, pricing, documentation, FAQ, blog, and comparison pages
* Whether the architecture clearly communicates topical authority

Provide recommendations for an improved page hierarchy and internal-linking strategy.

### 6. Structured data

Find all existing structured data and validate:

* JSON-LD syntax
* Schema type selection
* Required and recommended properties
* Consistency with visible page content
* Duplicate or conflicting schema
* Organization schema
* WebSite schema
* WebPage schema
* SoftwareApplication or Product schema, where appropriate
* Offer and pricing information
* BreadcrumbList
* FAQPage, only where content and current search-engine policies support it
* Article or BlogPosting
* Person and author information
* Review or AggregateRating eligibility
* ContactPoint
* SameAs references
* Logo, image, date, publisher, and author properties

Do not recommend misleading, invisible, or unsupported structured data.

For every schema recommendation, explain:

* Which pages should use it
* Why it is appropriate
* Which properties are required
* What project data should populate it
* Any eligibility or policy risks

### 7. AEO readiness

Assess how well the website is prepared for:

* Featured snippets
* People Also Ask results
* Voice and conversational search
* Direct answers
* Definition queries
* How-to queries
* Comparison queries
* Problem-solution searches

Check whether content includes:

* Clear question-based headings
* Concise direct answers
* Definitions
* Ordered steps
* Unordered lists
* Comparison tables
* FAQs
* Examples
* Use cases
* Supporting context after the direct answer

Recommend exact content formats and page locations, but do not rewrite the entire site during this audit.

### 8. GEO and AI citation readiness

Evaluate whether AI search systems and assistants can clearly understand, retrieve, verify, and cite the website.

Check:

* Clear organization and product identity
* Consistent entity naming
* Clear explanation of what the product does
* Specific, verifiable claims
* First-party facts, statistics, examples, case studies, and documentation
* Author, publisher, and company attribution
* Publication and update dates
* Source transparency
* About, contact, privacy, terms, and editorial information
* Consistent company details across the site
* Crawl access for relevant AI user agents
* Content availability in rendered HTML
* Stable, descriptive URLs
* Citation-friendly paragraphs
* Self-contained answers
* Unique first-party information
* Documentation and changelog quality
* External authority signals and SameAs references
* Whether key claims are supported by evidence
* Whether content appears promotional without enough factual substance

Review whether files or mechanisms such as `llms.txt` would add meaningful value for this project. Do not recommend them automatically without explaining their limitations and current practical value.

Identify why an AI assistant might avoid citing the website and what changes could make the site a stronger primary source.

### 9. Performance and technical quality

Review SEO-impacting technical factors, including:

* Core Web Vitals risks
* Server-side rendering and static generation
* Hydration issues
* Render-blocking resources
* JavaScript bundle size
* Font loading
* Image optimization
* Lazy loading
* Layout shifts
* Largest Contentful Paint
* Interaction responsiveness
* Caching
* Compression
* Preloading and preconnect usage
* Mobile responsiveness
* Accessibility problems that also affect SEO
* Broken components or console errors
* Error pages and empty states

Separate confirmed issues from potential risks that require live performance testing.

### 10. Analytics and search-platform setup

Check whether the project properly supports:

* Google Analytics
* Google Search Console verification
* Bing Webmaster Tools verification
* Consent handling, where required
* Conversion tracking
* UTM handling
* Event tracking
* Duplicate page-view prevention
* Production-only analytics
* Environment-variable safety

Do not expose secret values in the report.
